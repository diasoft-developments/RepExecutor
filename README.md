# RepExecutor

Проект для автоматического выполнения отчетов Diasoft на основе WRD-шаблонов.

## Описание

Скрипт автоматически обрабатывает отчеты в формате WRD (Delphi DFM), извлекает SQL-запросы, выполняет их на сервере Microsoft SQL Server, сохраняет результаты в CSV-файл для слияния данных, выполняет mail merge с шаблоном Microsoft Word и сохраняет итоговый отчет в формате PDF.

## Основные этапы работы

1. **Чтение конфигурации**: Загружает параметры из `config.json`, включая имя отчета, параметры замены и опциональные метки (marks).

2. **Поиск файлов**: На основе имени отчета ищет соответствующие WRD-файлы в базе данных.

3. **Парсинг WRD**: Извлекает SQL-запрос и имя документа из WRD-файла.

4. **Обработка SQL**: Заменяет параметры, добавляет команды для меток (если заданы), сохраняет полный SQL в файл.

5. **Выполнение SQL**: Запускает запрос на MS SQL Server, получает данные.

6. **Сохранение данных**: Записывает результаты в CSV-файл с заголовками.

7. **Mail Merge**: Выполняет слияние данных с Word-шаблоном и сохраняет результат в PDF.

## Конфигурация

Файл `config.json` содержит:
- `report_name`: Имя отчета для поиска файлов.
- `params`: Параметры для замены в SQL (например, `%Param!`).
- `marks` (опционально): Метки для добавления в tDocMark.

## Зависимости

- Python 3.x
- pyodbc (для MS SQL)
- pywin32 (для Word automation)
- loguru (для логирования)
- python-dotenv (для переменных окружения)

## Запуск

1. Настройте переменные окружения в `.env` (DB_SERVER, DB_DATABASE, DB_USER, DB_PASSWORD, DB_DRIVER).
2. Запустите `python main.py`.

## Логирование

Все действия логируются с помощью loguru. Уровни: INFO, WARNING, ERROR.
